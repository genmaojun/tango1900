## 2026-06-23 — v5.1（フォント方針・建築システム・デバッグ7点）

### フォント方針（可読性 × マイクラ感の両立）
- 学習で実際に読む文字（英単語・例文・日本語訳）だけシステムフォント、GUI 見出し・ボタン・数字・ロゴはピクセル（`PixelMC`/`JpDot`）維持、という切り分けが本プロジェクトの方針。
- 技術メモ：`<button>` は標準でフォントを継承しない。GUI をピクセルに戻す際、フォント指定を消すと選択肢ボタンが意図せず Arial などに落ちるので、フォントは明示する。
- 検証は `getComputedStyle` での実測が確実（スクショより正確）。

### Service Worker キャッシュ（最重要の再発防止）
- CSS/JS/フォントを変えたら `build.py` の `CACHE='tango1900-vNN'` を必ず +1。上げないと既存 PWA が古い版のまま。今回 v14→v15。
- プレビュー環境も SW キャッシュが効くため、確認前に SW unregister ＋ caches 削除 → reload が必要。

### デバッグ7視点（今後デフォルト）
静的／機能・正常系／表示・UI／異常系・境界値（連打含む）／回帰（既存データ互換）／非機能（オフライン・速度）／実機・環境差（iPhone Safari）。
配布前は実機・オフライン・記録互換・SW 版を特に確認する。「視点の数」より「壊れて困る順に潰す」。

### 建築システム（ゲーミフィケーション）
- 正解（〇ゲット）で `game.built` +1 ＝ブロック採掘。`BUILDS` 11段階で建造物が成長（テント→…→エンドポータル）。`buildOf()` で段階判定、到達で `buildComplete()` 演出＋レベルアップ音。
- `game` への新フィールドは `Object.assign({...,built:0}, readJSON())` と `game.built||0` で既存ユーザーの後方互換を確保（回帰の要点）。

## 2026-06-23 — 全テーマ文字コントラスト見直し（SW v15→v17）

「単語（例文ハイライト）が赤くて見にくい」という指摘を起点に、全5テーマ（volley/dark/light/minecraft/minecraft2）の**文字色 × 背景**を WCAG コントラスト比で総点検し、不足を是正した。

### 監査は「目」でなく「式」で
- 相対輝度 → コントラスト比をスクリプトで一括計算（`(L明+0.05)/(L暗+0.05)`）。スクショ目視より確実で、しきい値（本文4.5:1／大字・太字3:1）で機械的に判定できる。
- 失敗は **`--card2` に乗る文字**に集中していた。`--card2` は各テーマで「いちばん明るい面（暗色テーマ）／ほぼ白（light）」＝中間色の文字と輝度が近く、最も埋もれやすい面。

### テーマで“逆方向”に転ぶ
- **light**：文字が背景（白）に近すぎる → `--muted/--green/--red/--gold` を**濃色化**。
- **minecraft/minecraft2**：暗い文字（暗赤 `--red:#c0392b` 等）が中間色の茶/緑カードに埋もれる → ハイライト・スタッツ数字・苦手マークを**明色化**。
- 同じ「コントラストを上げる」でも方向が逆。背景の輝度を見て決める。

### トークン共有の罠（最重要の設計知見）
- 1つのトークンが「**白文字を載せる塗り**」と「**card2 上の文字**」の両用だと、暗色テーマで要求が逆になり**トークン値だけでは両立できない**。
  - 例：`--green` は `.ans.o`（白文字ボタン＝緑を暗くしたい）と `.stat.o b`（茶card2上の緑文字＝緑を明るくしたい）で対立。
  - 対策：トークンは塗り基準で据え置き、**文字側を要素別オーバーライド**（`html[data-theme^="minecraft"] .stat.o b{color:#b4ec8a}` 等）。
- 逆に **light は対立しない**（白文字ボタンも card2 上文字も「濃くする」で一致）ので、トークン値変更だけで両方解決できた。

### amber＋白は全テーマで最悪 → 暗文字へ
- 回答ボタン `.ans.d`（amber）の白文字は 1.5〜2.1:1。amber は明るいので**白でなく濃茶文字**が正解（警告色＝暗文字on黄、の一般則）。amber 自体はボタンの明るさを保ち、light のスタッツ数字だけ濃ゴールドで個別上書き。
- ○緑/×赤ボタンの白文字は大字・太字・1単語で慣例どおり維持（実用上読める）。

### `auto` テーマの分岐に注意
- `light` 向けの文字オーバーライドを無条件に `auto` へ広げると**ダークモードのauto**に漏れて破綻する。`@media (prefers-color-scheme:light){ html[data-theme="auto"] … }` で**light時のみ**に限定する。

### 再発防止
- 配色や文字色を触るときは、新色を `(L明+0.05)/(L暗+0.05)` で**必ず数値検証**してからコミット。`--card2` 上の文字を最優先で確認する。
- CSS 変更につき SW を **v15→v16（ハイライト）→v17（全体）** と都度 +1（上げ忘れ＝既存PWAが旧版固定）。
